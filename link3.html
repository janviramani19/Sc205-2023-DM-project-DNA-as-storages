♦ Cyclic Hamming Codes : 

            → If g(x) is a primitive polynomial of degree m over GF(2), the ring of polynomials modulo g(x), GF(2)/g(x), is the field of order 2m.

            → Let α be a root of g(x), then {0,1,α,α2,....,α2^m-2} are the 2m elements of the field.Each element can also be represented by a binary m-tuple.

            → Use the (2m-1) non-zero elements to construct the columns of a matrix 
                H= [0,1,α,α2,....,α2^m-2]

            → The code C with parity check matrix H is a Hamming code with n=2m-1 as H contains all district non-zero m- tuples .

            → Since cHt=0, we can express the codewords as 
                    C ={c0, c1,....cn-1 | c0+c1 x α + c2 x α2+....cn-1 x αn-1}
                    -> c(x) has root a since c(a)=0

            → As g(a)=0, c(x) is a multiple of g(x)
                    - therefore c(x) is a cyclic code .
                    - all binary Hamming codes are equivalent to cyclic codes .

            → Example: g(x)= x3+ x+1 -> GF(2)[x]/g(X) is GF(8)
                            The field elements are 

                    {0,1,α,α2,α3=α+1,α4=α2+α,α5=α2+α+1,α6=α2+1}

♦ BCH Codes: 

            → Binary BCH coes are a generalization of cyclic Hamming codes

            -g(x)is a primitive polynomial
            -c(α)=0 if α is a root of g(x) 
            - the corresponding parity check matrix has columns corresponding to all non-zero powers of α or all distinct non-zero m-tuples.

♦ Golay Subcode Model: 

            → In Golay code changes of string into ACGT DNA 
            → There are some steps to convert this strings into ACGT:
                    1. Each character need to convert into ASCII value
                    2. Each ASCII value need to convert into ternary language.
                    3. Then convert each ternary into ACGT string via the table below:
0 1 2
T A C G
G T A C
C G T A
A C G T
